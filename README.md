# ServerVault — AI-Generated Server Inventory Manager

ServerVault is a local, self-hosted server inventory manager designed for security, portability, and ease of use. It features a modern dark-themed UI, multi-factor authentication (2FA), and a powerful Bearer Token API for external integrations (n8n, CI/CD, scripts).

## 🚀 Quick Start

### Prerequisites
- Node.js 18+
- PostgreSQL 15+ (or Docker)
- Docker & Docker Compose (optional)
- Make (optional)

### Setup & Installation
1.  **Clone & Install:**
    ```bash
    make install
    ```
2.  **Environment Variables:**
    Configure your `.env` file in the `server` directory:
    ```env
    POSTGRES_HOST=localhost
    POSTGRES_PORT=5432
    POSTGRES_USER=postgres
    POSTGRES_PASSWORD=postgres
    POSTGRES_DATABASE=servervault
    SESSION_SECRET=your-secret-key
    ```
3.  **Initialize Database:**
    ```bash
    make seed
    ```
3.  **Start Development Mode:**
    ```bash
    make dev
    ```
    - Frontend: `http://localhost:5173`
    - Backend: `http://localhost:3000`

### Docker Deployment
```bash
make start
```

## 🔐 Authentication

### First Login
Use the following credentials to access the system:
- **Email/Username:** `admin`
- **Password:** `password123`

### 2FA Setup
1. Go to the **Profile** page.
2. Click **Enable 2FA**.
3. Scan the QR code with Google Authenticator or Authy.
4. Enter the 6-digit code to verify.

### API Bearer Tokens
Generate tokens in the **Profile** page for external access.
```bash
curl -H "Authorization: Bearer svk_your_token" http://localhost:3000/api/servers
```

## 📊 Features
- **Dashboard:** Real-time stats and resource distribution charts.
- **Server Inventory:** Detailed node management with right-side drawer and 3-step wizard.
- **Organization:** Manage Groups, Tags, and SSH Key metadata.
- **Data Portability:** Export/Import entire database as JSON or CSV.
- **Security:** Session-based Web UI + Hashed Bearer Tokens + TOTP 2FA.

## 🤖 n8n Integration Guide
1. Create a new **Header Auth** credential in n8n.
2. Set Name to `Authorization` and Value to `Bearer svk_xxx`.
3. Use the **HTTP Request** node to interact with `/api/v1/servers`.
